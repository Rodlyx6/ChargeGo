## 🎯 完整业务流程

### 用户充电完整流程

```
1. 注册/登录
   ↓
2. 搜索附近充电桩
   - 输入经纬度或使用当前位置
   - 设置搜索半径
   - 查看充电桩列表（距离、状态）
   ↓
3. 选择充电桩并预约
   - 点击"立即预约"
   - 确认预约信息
   - 系统锁定充电桩（状态变为"预约中"）
   - 生成订单（状态为"待支付"）
   - 发送15分钟超时消息
   ↓
4. 支付订单
   - 进入"我的订单"页面
   - 查看待支付订单
   - 点击"立即支付"
   - 订单状态变为"充电中"
   - 充电桩状态变为"充电中"
   ↓
5. 充电过程
   - 订单状态显示"充电中"
   - 可以随时结束充电
   ↓
6. 结束充电
   - 点击"结束充电"
   - 订单状态变为"已完成"
   - 充电桩状态变为"空闲"
   ↓
7. 查看历史订单
   - 查看所有订单记录
   - 查看订单详情
```

### 超时取消流程

```
1. 用户预约充电桩
   ↓
2. 系统发送延迟消息到 RabbitMQ（TTL=15分钟）
   ↓
3. 15分钟后消息进入死信队列
   ↓
4. 监听器接收消息
   ↓
5. 检查订单状态
   - 如果仍为"待支付" → 自动取消
   - 如果已支付 → 不处理
   ↓
6. 释放充电桩（状态变为"空闲"）
```

## 🔧 技术实现要点

### 1. 雪花算法 ID 精度问题解决
- **问题**：JavaScript Number 类型只能安全表示 53 位整数
- **解决**：
  - 后端：Long 序列化为字符串（JacksonConfig）
  - 前端：Axios 自动处理大数字（transformResponse）
  - 传输：始终以字符串形式传输 ID

### 2. JWT Token 认证
- **生成**：用户登录时生成 Token（有效期7天）
- **传输**：前端通过 Authorization 头发送
- **验证**：网关统一验证，提取用户信息
- **权限**：区分普通用户和管理员

### 3. 分布式锁
- **工具**：Redisson
- **场景**：预约充电桩时防止超卖
- **流程**：
  1. 获取锁（key: `lock:station:{stationId}`）
  2. 检查充电桩状态
  3. 锁定充电桩
  4. 创建订单
  5. 释放锁

### 4. 延迟消息（订单超时）
- **工具**：RabbitMQ 死信队列
- **流程**：
  1. 创建订单时发送消息到普通队列（TTL=15分钟）
  2. 15分钟后消息过期，转入死信队列
  3. 监听器消费死信队列
  4. 检查订单状态，自动取消未支付订单

### 5. 服务间调用
- **工具**：OpenFeign
- **场景**：
  - order-service → station-service（更新充电桩状态）
  - order-service → user-service（验证用户信息）
- **熔断**：Fallback 机制

## 📊 数据库表结构

### user 表
- id (BIGINT) - 主键，雪花ID
- phone (VARCHAR) - 手机号，唯一
- password (VARCHAR) - 密码，MD5加密
- nickname (VARCHAR) - 昵称
- role (TINYINT) - 角色（0-普通用户，1-管理员）
- create_time (DATETIME) - 创建时间
- update_time (DATETIME) - 更新时间

### station 表
- id (BIGINT) - 主键，雪花ID
- sn_code (VARCHAR) - 设备编号，唯一
- status (TINYINT) - 状态（0-空闲，1-预约中，2-充电中，3-故障）
- address (VARCHAR) - 详细地址
- location (POINT) - 经纬度，空间索引
- create_time (DATETIME) - 创建时间
- update_time (DATETIME) - 更新时间

### order_info 表
- id (BIGINT) - 主键，雪花ID
- order_no (VARCHAR) - 订单号，唯一
- user_id (BIGINT) - 用户ID
- station_id (BIGINT) - 充电桩ID
- status (TINYINT) - 状态（0-待支付，1-充电中，2-已完成，3-已取消）
- pay_time (DATETIME) - 支付时间
- create_time (DATETIME) - 创建时间
- update_time (DATETIME) - 更新时间
