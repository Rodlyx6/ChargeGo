# ChargeGo 共享充电桩平台 - 需求规格说明书

## 1. 项目概述

### 1.1 项目名称
ChargeGo 共享充电桩平台

### 1.2 项目背景
随着新能源汽车的快速普及，充电基础设施的需求日益增长。然而，当前存在以下问题：
- **找桩难**：用户难以快速找到附近可用的充电桩
- **排队久**：到达充电桩后发现已被占用，需要排队等待
- **信息不对称**：充电桩状态信息不透明，用户无法实时了解

### 1.3 项目目标
设计并实现一个共享充电桩平台，通过以下功能解决上述问题：
1. 实时查询附近充电桩状态
2. 一键预约锁桩，避免排队
3. 在线支付，快速开始充电
4. 自动计费，灵活结束充电
5. 管理员后台管理充电桩和订单

### 1.4 项目范围
- **用户端**：普通用户的充电预约和支付功能
- **管理端**：管理员的充电桩和订单管理功能
- **后端服务**：微服务架构，包括用户、充电桩、订单三个核心服务
- **数据库**：MySQL 数据库，存储用户、充电桩、订单等数据

## 2. 功能需求

### 2.1 用户端功能

#### 2.1.1 用户认证
- **用户注册**
  - 输入：手机号、密码、昵称
  - 验证：手机号唯一性、密码强度
  - 输出：注册成功/失败提示

- **用户登录**
  - 输入：手机号、密码
  - 验证：账号密码正确性
  - 输出：JWT Token、用户信息

- **用户登出**
  - 清除本地 Token 和用户信息

#### 2.1.2 充电桩搜索
- **附近搜索**
  - 输入：当前位置（经纬度）、搜索半径
  - 处理：基于地理位置的距离计算
  - 输出：附近充电桩列表（包含距离、状态、地址等）

- **充电桩详情**
  - 显示：充电桩编号、地址、状态、距离等信息

#### 2.1.3 充电预约
- **选择充电类型**
  - 快充（10元/小时）
  - 普通充电（5元/小时）
  - 慢充（2元/小时）

- **选择充电时长**
  - 支持 1-4 小时选择
  - 实时显示预期费用

- **预约确认**
  - 锁定充电桩（15分钟内必须支付）
  - 生成订单号

#### 2.1.4 订单管理
- **订单列表**
  - 显示所有订单（待支付、充电中、已完成、已取消）
  - 按时间倒序排列

- **订单详情**
  - 显示：订单号、充电类型、预约时长、预期费用、实际费用、退款金额等
  - 动态显示：支付倒计时（待支付）、充电时长（充电中）、完单时间（已完成）

- **订单操作**
  - 待支付：支付订单、取消订单
  - 充电中：结束充电
  - 已完成：查看退款信息

#### 2.1.5 支付功能
- **在线支付**
  - 支付订单（将订单状态从待支付改为充电中）
  - 支付超时自动取消（15分钟）

- **费用计算**
  - 预期费用 = 充电类型单价 × 预约时长
  - 实际费用 = 充电类型单价 × 实际充电时长
  - 退款金额 = 预期费用 - 实际费用

### 2.2 管理员端功能

#### 2.2.1 管理员认证
- **管理员登录**
  - 输入：手机号、密码、勾选"管理员登录"
  - 验证：账号密码正确性、角色为管理员
  - 输出：JWT Token、管理员信息

#### 2.2.2 充电桩管理
- **充电桩列表**
  - 显示所有充电桩（空闲、预约中、充电中、故障）
  - 支持分页查询

- **新增充电桩**
  - 输入：充电桩编号、地址、经纬度
  - 验证：编号唯一性
  - 输出：新增成功/失败提示

- **编辑充电桩**
  - 修改：地址、经纬度等信息
  - 验证：编号不可修改

- **删除充电桩**
  - 删除指定充电桩

- **充电桩统计**
  - 显示：总数、空闲数、预约中数、充电中数、故障数

#### 2.2.3 订单管理
- **订单列表**
  - 显示所有订单
  - 支持按状态、用户、时间筛选
  - 支持分页查询

- **订单详情**
  - 显示：订单号、用户信息、充电桩信息、充电类型、预约时长、费用等

- **订单统计**
  - 显示：总订单数、待支付数、充电中数、已完成数、已取消数
  - 显示：总收入、平均订单金额等

#### 2.2.4 用户管理
- **用户列表**
  - 显示所有用户
  - 支持按角色、注册时间筛选
  - 支持分页查询

- **用户详情**
  - 显示：用户ID、手机号、昵称、角色、注册时间等

- **用户统计**
  - 显示：总用户数、普通用户数、管理员数

## 3. 非功能需求

### 3.1 性能需求
- **响应时间**：API 响应时间 < 500ms
- **并发处理**：支持 1000+ 并发用户
- **数据库查询**：单次查询时间 < 100ms

### 3.2 安全需求
- **身份认证**：使用 JWT Token 进行身份认证
- **权限控制**：基于角色的访问控制（RBAC）
- **数据加密**：密码使用 BCrypt 加密存储
- **API 安全**：使用 HTTPS 传输，防止中间人攻击

### 3.3 可用性需求
- **系统可用性**：99.9% 以上
- **故障恢复**：支持自动故障转移和恢复
- **数据备份**：每日自动备份数据库

### 3.4 可扩展性需求
- **微服务架构**：支持服务独立扩展
- **数据库分片**：支持数据库水平扩展
- **缓存机制**：使用 Redis 缓存热数据

### 3.5 用户体验需求
- **界面友好**：简洁直观的用户界面
- **响应及时**：实时显示充电桩状态、订单信息
- **错误提示**：清晰的错误提示和操作指导

## 4. 业务流程

### 4.1 用户充电流程
```
1. 用户注册/登录
   ↓
2. 搜索附近充电桩
   ↓
3. 选择充电桩
   ↓
4. 选择充电类型和时长
   ↓
5. 确认预约（锁定充电桩）
   ↓
6. 支付订单（15分钟内）
   ↓
7. 开始充电
   ↓
8. 结束充电（自动计费）
   ↓
9. 查看订单详情和退款信息
```

### 4.2 管理员管理流程
```
1. 管理员登录
   ↓
2. 查看管理概览（统计数据）
   ↓
3. 管理充电桩（新增、编辑、删除）
   ↓
4. 查看订单列表和详情
   ↓
5. 查看用户列表和详情
```

## 6. 约束条件

### 6.1 技术约束
- 后端：Java Spring Boot 微服务架构
- 前端：Vue 3 + Vite
- 数据库：MySQL 8.0+
- 缓存：Redis 6.0+
- 消息队列：RabbitMQ

### 6.2 业务约束
- 预约后 15 分钟内必须支付，否则自动取消
- 充电类型和价格由管理员配置
- 用户只能有一个待支付的订单

### 6.3 法律约束
- 遵守数据保护法规
- 用户隐私保护
- 支付安全合规
